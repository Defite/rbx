# Form Controls

All generic **form controls**, designed for consistency.

### Fields

The `<Field>` component is a container for a `<Label>`, a `<Control>` and optional `<Help>` text.

<Playground>
  <Field>
    <Label>Label</Label>
    <Control>
      <Input type="text" placeholder="Text input" />
    </Control>
    <Help>This is a help text</Help>
  </Field>
</Playground>

Notice that the `<Field>` container allows for consistent spacing.

<Playground>
  <Field>
    <Label>Name</Label>
    <Control>
      <Input type="text" placeholder="e.g. Alex Smith" />
    </Control>
  </Field>

  <Field>
    <Label>Email</Label>
    <Control>
      <Input type="email" placeholder="e.g. alexsmith@gmail.com" />
    </Control>
  </Field>
</Playground>

### Controls

The `<Control>` component enhances a **single** form control.
It has the same height as a control element and can _only_ contain an `<Input>`, a `<Select>`, a `<Button>` and an `<Icon>`.

<Playground>
  <Control>
    <Input type="text" placeholder="Text input" />
  </Control>
</Playground>

<Playground>
  <Select.Container>
    <Select>
      <Select.Option value="dropdown">Select dropdown</Select.Option>
      <Select.Option value="options">With Options</Select.Option>
    </Select>
  </Select.Container>
</Playground>

<Playground>
  <Button color="primary">Submit</Button>
</Playground>

### Fields with addons

Use the `kind` prop as `addons` on a `<Field>` component to attach controls together.

<Playground>
  <Field kind="addons">
    <Control>
      <Input placeholder="Find a repository" />
    </Control>
    <Control>
      <Button color="info">Search</Button>
    </Control>
  </Field>
</Playground>

You can only attach `<Input>`s, `<Button>`s and `<Select>`s.

<Playground>
  <Field kind="addons">

### Form group

Use the `kind` prop as `group` on `<Field>` to group controls together, and use the `align` prop on `<Field>` to alter the alignment.

<Playground>
  {() =>
    [undefined, ...FIELD_DEFAULTS.alignments].map((align, i) => (
      <OptionBlock props={{ align }} index={i} key={i}>
        <Field kind="group" align={align}>
          <Control>
            <Button color="primary">Submit</Button>
          </Control>
          <Control>
            <Button color="light">Cancel</Button>
          </Control>
        </Field>
      </OptionBlock>
    ))
  }
</Playground>

Use the `expanded` prop on the `<Control>` to fill up the remaing space.

<Playground>
  <Field kind="group">
    <Control expanded>
      <Input placeholder="Find a repository" />
    </Control>
    <Control>
      <Button color="info">Search</Button>
    </Control>
  </Field>
</Playground>

Use the `mutliline` prop on `<Field>` to fill up multiple lines (ideal for a long list of controls).

### Form layouts

Use the `horizontal` prop of a `<Field>` for horizontal form controls.

You can still use the `kind` property on `<Field>`, but you should use `<Field.Label>` for the side label and `<Field.Body>` for the `<Input>`, `<Select>` or `<Textarea>` components.

### Disabled fields

Use the `disabled` prop on `<Fieldset>` to set all child fields as disabled.

<Playground>
  <Fieldset disabled>
    <Field>
      <Label>Name</Label>
      <Control>
        <Input type="text" placeholder="e.g. Alex Smith" />
      </Control>
    </Field>
    <Field>
      <Label>Email</Label>
      <Control>
        <Input type="email" placeholder="e.g. alexsmith@gmail.com" />
      </Control>
    </Field>
  </Fieldset>
</Playground>
