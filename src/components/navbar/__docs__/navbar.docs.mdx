# Navbar

A responsive horizontal **navbar** that can support images, links, buttons, and dropdowns.

### Navbar brand

The `<Navbar.Brand>` is on the left side of the `<Navbar>`.
It can contain `<Navbar.Item>`s and, as it's last child, a `<Navbar.Burger>`.

It is **always visible**, so you should only use a few `<Navbar.Item>`s to avoid overflowing horizontally on small **touch** devices.

On **desktop** `>=1024px`, the `<Navbar.Brand>` will only take up the space it needs.

<Playground>
  <Navbar>
    <Navbar.Brand>
      <Navbar.Item href="https://bulma.io">
        <img
          src="https://bulma.io/images/bulma-logo.png"
          alt="Bulma: Free, open source, & modern CSS framework based on Flexbox"
          width="112"
          height="28"
        />
      </Navbar.Item>
      <Navbar.Burger />
    </Navbar.Brand>
  </Navbar>
</Playground>

<Notification>
  Resize your browser window to see the <code>{"<Navbar.Burger>"}</code> appear.
</Notification>

### Navbar burger

The `<Navbar.Burger>` is a hamburger menu that appears only on **touch** devices.
It must be the last child of the `<Navbar.Brand>`.

### Navbar menu

The `<Navbar.Menu>` is a **counterpart** of the `<Navbar.Brand>`.
It must appear as a direct child of `<Navbar>`, and as a sibling of `<Navbar.Brand>`.

It is hidden on **touch** devices `<1024px`. Though you can use the `active` and `managed` props of `<Navbar>` to force it open or closed.

On **desktop** and bigger devices `>=1024px`, the `<Navbar.Menu>` will **fill up the sapce** available in the `<Navbar>`, leaving the `<Navbar.Brand>` just the space it needs.

It needs two other components as children: `<Navbar.Segment align="start">` and `<Navbar.Segment align="end">`.

### Start and end

Use the `<Navbar.Segment>` component and its `align` prop as the direct (and only) descendents of `<Navbar.Menu>`.

Each `<Navbar.Segment>` can contain any number of `<Navbar.Item>`s.

This will automatically add the `has-navbar-fixed-top` or `has-navbar-fixed-bottom` classes to the Document supplied in the `document`, or the Document upon which your app is mounted.
This is useful for mounting in `iframe`s.

### Dropdown menu

To create a **\*dropdown menu** you'll need the following **5** components:

- `<Navbar.Item>` with the `dropdown` prop
- `<Navbar.Link>` which contains the dropdown arrow
- `<Navbar.Dropdown>` which contains instances of `<Navbar.Item>` and `<Navbar.Divider>`

<Playground>
  <div style={{ height: "200px " }}>
    <Navbar>
      <Navbar.Item dropdown>
        <Navbar.Link>Docs</Navbar.Link>
        <Navbar.Dropdown>
          <Navbar.Item>Overview</Navbar.Item>
          <Navbar.Item>Elements</Navbar.Item>
          <Navbar.Item>Components</Navbar.Item>
          <Navbar.Divider />
          <Navbar.Item>Version 2.0.0</Navbar.Item>
        </Navbar.Dropdown>

</Playground>
